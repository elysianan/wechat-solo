# WeChat Solo · 演示录屏脚本

> 用途：作品集演示视频 / 面试现场演示提词。总时长约 3.5 分钟。
> 录制前：`npm install && npm run dev`，浏览器打开 http://localhost:5173

---

## 录制准备

1. **窗口比例**：项目容器最大宽度 430px，浏览器窗口调窄（或用开发者工具设备模拟选 iPhone 尺寸），让手机壳居中。
2. **数据状态**：如需干净的演示数据，先在开发者工具 Application → IndexedDB 删除 `WeChatSoloDB` 后刷新，会重新载入种子数据。
3. **节奏**：每个镜头操作后停顿 1~2 秒，给观众看清的时间；Agent 回复有随机延迟（1~3 秒），正好用来讲台词。
4. **浅色录主流程，深色模式留作高潮镜头**（第 7 镜）。

---

## 分镜脚本

### 第 1 镜：开场 + 聊天列表（0:00–0:20）

| 项 | 内容 |
|----|------|
| 画面 | 应用首页，聊天列表：5 个会话、头像、未读角标、最后消息预览 |
| 操作 | 不动，直接开讲 |
| 讲解词 | 「这是 WeChat Solo——一个纯前端、可离线运行的微信模拟器。里面所有好友都不是真人，而是我设计的 AI Agent 人设。」 |

### 第 2 镜：Agent 聊天（0:20–1:00）

| 项 | 内容 |
|----|------|
| 画面 | 点击进入「王阿姨」会话 |
| 操作 | 输入「妈，我这个月加班比较多」，发送 |
| 讲解词 | 「给妈妈发条消息。注意看——先出现『对方正在输入』，停顿一两秒后回复才到。她的人设是唠叨、节俭的妈妈，所以回复会很长、带很多叮嘱和表情。」 |
| 备注 | 等回复到达后继续；可指着消息旁的双绿勾说「消息状态也做了：单灰勾、双灰勾、双绿勾，已读自动触发」 |

### 第 3 镜：换个 Agent 对比人设（1:00–1:25）

| 项 | 内容 |
|----|------|
| 画面 | 返回列表，进入「张总」会话 |
| 操作 | 输入「张总，方案发您了」，发送 |
| 讲解词 | 「再对比一下老板——注意区别：张总从不显示『正在输入』，因为人设里他的打字提示概率是零；回复要等几秒，但内容短促、命令式，常用『收到』。五个角色各有独立的行为参数：回复延迟、打字概率、甚至已读不回的概率都不同。」 |

### 第 4 镜：通讯录与拼音搜索（1:25–1:50）

| 项 | 内容 |
|----|------|
| 画面 | 切到「通讯录」tab |
| 操作 | 搜索框输入「wang」→ 命中王阿姨；清空后点右侧字母索引跳转 |
| 讲解词 | 「通讯录支持拼音搜索——输入拼音首字母就能命中中文名字；右侧字母索引可以跳转分组。好友资料页点头像进入，可以直接发消息。」 |

### 第 5 镜：朋友圈（1:50–2:20）

| 项 | 内容 |
|----|------|
| 画面 | 「发现」→「朋友圈」 |
| 操作 | 给一条动态点赞、写一条评论 |
| 讲解词 | 「朋友圈的点赞和评论都是真实写入浏览器 IndexedDB 的——刷新页面数据还在。整个应用没有任何后端，数据全部存在本地。」 |

### 第 6 镜：个人信息编辑（2:20–2:45）

| 项 | 内容 |
|----|------|
| 画面 | 「我」tab → 点击信息卡 |
| 操作 | 点「昵称」→ 改成新名字 → 保存 → 返回「我」页 |
| 讲解词 | 「个人信息可以编辑并持久化。改完昵称返回，卡片上立即更新，刷新也不会丢。」 |

### 第 7 镜：深色模式（高潮镜头，2:45–3:10）

| 项 | 内容 |
|----|------|
| 画面 | 「我」→「设置」→ 打开「深色模式」开关 |
| 操作 | 拨动开关后，返回「我」页，再切到聊天列表展示换肤效果 |
| 讲解词 | 「最后是深色模式。它没有用常规的逐元素适配，而是 CSS 变量主题——切换瞬间整个应用换肤，包括前面看过的所有页面，而且设置同样持久化。」 |

### 第 8 镜：收尾（3:10–3:30）

| 项 | 内容 |
|----|------|
| 画面 | 设置 → 关于 WeChat Solo（版本号）；最后画面停在任意页面，露出底部水印 |
| 操作 | 指向屏幕底部水印 |
| 讲解词 | 「项目从产品设计、Agent 人设到代码和测试全部独立完成，5 个 Sprint、119 个测试用例，代码已开源在 Gitee。这个水印也说明了它的定位：一个用于学习和展示的作品。谢谢观看。」 |

## 备选镜头（时间充裕或面试官指定时加）

- **支付页**：「我」→「支付」→ 绿色余额卡 + 六宫格入口（30 秒）。话术：「支付是明确降级的 Mock 功能——余额是演示数据，入口点击会提示演示模式。这体现了产品边界意识：核心 IM 体验做真，敏感功能只做 UI。」
- **已读不回**：给「Lisa」发消息，有一定概率触发已读不回（消息变双绿勾但无回复）。话术：「暧昧同事的人设里配置了已读不回概率——社交产品里沉默也是体验的一部分。」
